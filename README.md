# Оплати медичним закладам за програмою медичних гарантій (2024)

**Мета:** Проаналізувати, як держава фінансувала медичні заклади у 2024 році, дослідити різницю у виплатах між закладами та розподіл коштів у часі.
**Джерело даних:** [Портал відкритих даних (data.gov.ua)](https://data.gov.ua/dataset/25a46db9-2f15-4302-9b59-9bd761c80f46)

**Гіпотези для аналізу:**
1. Нерівномірність виплат: Більшість коштів отримує мала кількість великих установ.
2. Типи власності: Виплати ФОП значно нижчі, ніж державним КНП.
3. Часова активність: Виплати мають пікові дні.
*Якість даних перевірено. Аналіз підтвердив усі висунуті гіпотези.*

---

## Контейнерна архітектура (Docker)

### 1. Структура проєкту
* `data/raw/` — директорія з вхідним файлом `payments_2024.csv`.
* `data_load/` — скрипти та Dockerfile для завантаження даних у БД.
* `quality_analysis/` — скрипти та Dockerfile для аналізу якості даних.
* `data_research/` — скрипти та Dockerfile для перевірки гіпотез.
* `visualization/` — скрипти та Dockerfile для побудови графіків.
* `web/` — Flask-додаток (код та шаблони HTML) для веб-інтерфейсу.
* `compose.yaml` — головний файл оркестрації мікросервісів.

### 2. Опис сервісів
* **db (PostgreSQL):** Єдине джерело сирих даних. Використовує том `pgdata` для збереження стану.
* **data_load:** Ініціалізує таблицю і завантажує дані з CSV у PostgreSQL. Вимикається після виконання.
* **quality_analysis**, **data_research**, **visualization:** Аналітичні сервіси. Запускаються паралельно після завантаження даних. Читають БД, обчислюють результати та зберігають звіти (txt, json) і графіки (png) у спільний Docker-том `shared_data`.
* **web:** Веб-сервер. Запускається останнім, агрегує артефакти зі спільного тому та виводить дашборд у браузер.

### 3. Перелік портів
* **5432** — База даних PostgreSQL (доступна для підключення інструментами аналізу).
* **5000** — Веб-інтерфейс Flask.
